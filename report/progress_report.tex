\section{Brief Project Description}
\vspace{-4pt}

Syntax highlighters, code search engines, snippet managers and Q\&A sites such as Stack Overflow
must constantly identify a block of code's language, often from a short fragment with no filename
or extension \citep{alreshedy2018scc}. \textbf{This project maps a raw snippet to one of 10 popular
languages (Python, Java, C++, JavaScript, Go, C, Ruby, PHP, Rust, C\#) from its characters alone.}
Extensions and editor metadata are exactly what is missing in pasted snippets, chat messages and
scraped pages; rule-based detectors break on short fragments \citep{vandam2016software}, and
GitHub's Linguist leans on extensions \citep{linguist}, so neither says much about a bare
snippet---the gap a learned model fills.

Deep learning fits this input/output pair for three reasons. Languages differ by \emph{subtle,
overlapping} cues (keywords, operator spellings, bracket and indentation patterns) a
character-level CNN learns automatically rather than by hand-engineering features per language
\citep{zhang2015character}; those cues are \emph{local} n-gram motifs (\texttt{:=}, \texttt{fn},
\texttt{\$var}), which is exactly what small-kernel convolution detects
\citep{kim2014convolutional}; and character input side-steps a chicken-and-egg problem---a
token-based model needs a tokenizer, but choosing one presupposes knowing the language.
Figure~\ref{fig:pipeline} is the contract: \textbf{input} raw text padded/truncated to 256
characters, \textbf{output} a softmax over the 10 languages.

\centering

\vspace{-4pt}
\subsection{Data Processing}
\vspace{-4pt}
\textbf{Source.} Rosetta Code \citep{rosettacode} (\texttt{acmeism/RosettaCodeData} mirror)
implements the same tasks across many languages, so snippets are labelled by directory, not
guessed. Table~\ref{tab:clean} gives the full pipeline (\texttt{src/build\_dataset.py}, seed 360) and its
yield, specified so a classmate can reproduce it. Two steps carry most of the weight: \emph{near-duplicate removal} (MinHash/LSH over 5-character
shingles drops pairs with Jaccard $\geq0.8$ \citep{broder1997resemblance}---Rosetta hosts many
near-identical variants of one solution) and \emph{splitting by task, not by file}. Snippets are
emitted as \texttt{raw} and---where they still clear the 64-character floor---\texttt{nocomment}
(10{,}271), to test reliance on comment prose versus code structure.

\textbf{Example cleaned training sample}, verbatim from \texttt{raw\_train.jsonl} (class
\textsc{Python}, task \texttt{Call-a-foreign-language-function}, 121 characters; the two
\texttt{\#} comments are what the \texttt{nocomment} variant strips):
\begin{lstlisting}[language=Python]
import ctypes
libc = ctypes.CDLL("/lib/libc.so.6")
libc.strcmp("abc", "def")     # -1
libc.strcmp("hello", "hello") #  0
\end{lstlisting}

\textbf{Plan for never-before-seen data.} Rather than reuse a public dataset I collected the
final test set myself (\texttt{src/scrape\_heldout.py}): \textbf{740 snippets from 24 real GitHub repositories}, 74 per language, through the same
windowing, comment and exact-dedup steps (near-dup dedup is moot here---no two library files are
variants of one task). None is a Rosetta source, and it is deliberately a \emph{different
distribution}: production library code, not puzzle solutions. Each snippet is checked against the SHA-1 of every training
snippet---\textbf{0 leaked}---and is never trained or tuned on.

\textbf{Challenges.} (i) PHP is thin on Rosetta (645 files vs.\ Python's 3{,}054), so balancing
drags \emph{every} class to 1{,}036---the biggest limit on dataset size. (ii) I split by task expecting a per-snippet split to leak; measuring it
(\texttt{src/leakage\_experiment.py}) showed otherwise---it moves the baseline $+0.20$ and the CNN
$+0.00$, so near-dup removal, not the split, is what prevents leakage here. I kept the by-task
split as the safer default. (iii) The line-based stripper over-strips string literals containing \texttt{//} or \texttt{\#},
and for C deletes \texttt{\#include}/\texttt{\#define}, which are \emph{code}---biasing the
ablation \emph{against} the CNN's 92.49, so that conclusion is conservative. Fixed for the final.

\vspace{-6pt}
\subsection{Baseline Model}
\vspace{-4pt}

The baseline uses no deep learning: TF-IDF over \textbf{character n-grams} ($n\!\in\![1,3]$,
\texttt{min\_df}=2, sublinear tf, case preserved) into \textbf{Multinomial Naive Bayes} on the same
splits \citep{khasnabish2014detecting,ugurel2002whats}. It needs no tuning beyond the fixed n-gram
range, uses 43{,}573 features and trains in $\approx$4\,s. It is an honest bar, not a straw
man---\textbf{90.14\%} versus 10\% chance---so n-grams already carry most of the signal.

\textbf{Qualitative (baseline).} Its held-out errors are prose-driven: it labels PHP \textsc{C\#}
\textbf{34/74} times and gets PHP right only \textbf{26/74}---a pair it never confuses
in-distribution (0 cases)---and Java$\to$Python (24/74) and C$\to$Python (20/74) repeat the shape:
English comment blocks drag an n-gram model toward whichever class carries the most prose.
\textbf{Challenge:} no tuning fixes this---the failure is the representation, which is the gap the
CNN closes.

\vspace{-6pt}
\subsection{Primary Model}
\label{sec:primary}
\vspace{-4pt}

The primary model (Figure~\ref{fig:pipeline}) is \texttt{Embedding}$(102,32)$
(\texttt{padding\_idx}=0); three \emph{parallel} \texttt{Conv1d} branches of widths 3/5/7,
128 filters each $+$ ReLU; global max-pool per branch; concatenation to 384;
\texttt{Dropout}(0.5); \texttt{Linear}$(384,10)$ $+$ softmax. Parallel widths rather than a deep stack see 3-, 5- and 7-character motifs at once---the scale at
which \texttt{fn}, \texttt{:=} and \texttt{\$var} live. It has \textbf{68,938 trainable parameters} in 5 weight layers, trained from scratch with
cross-entropy and Adam \citep{kingma2015adam} (lr $10^{-3}$, batch 64, $\leq$30 epochs, early
stopping on validation accuracy, patience 6), in \textbf{35\,s} on an Apple M-series GPU (MPS).
Validation loss flattens near epoch 10 and the train/validation gap widens after epoch 12---mild
overfitting early stopping controls (stopped at 25, best \textbf{epoch 19}).

I also built the proposal's bidirectional GRU (\texttt{src/gru.py}; 42{,}186 parameters).
\textbf{It did not pan out}---the most useful negative result so far: 90.27\% test,
indistinguishable from Naive Bayes, for \textbf{647\,s of training, 18$\times$ the CNN}. Language
identity in a 256-character window rests on \emph{local} motifs, so recurrence buys nothing.

\textbf{Comment ablation.} Stripping comments costs the CNN only \textbf{0.25} (92.74$\to$92.49):
it reads \emph{code structure, not comment prose}. The baseline behaves oppositely---comments
removed, its held-out score \emph{rises} 2.30 (79.59$\to$81.89).

\textbf{Qualitative (CNN).} The 9.9-point gap between the two test sets (Figure~\ref{fig:cm}) is
the most interesting result so far, because the errors explain it. \emph{(1) Java$\to$JavaScript
(30 errors): licence headers.} \textbf{22 of the 30} carry an Apache/copyright header, averaging
only \textbf{17\% code} by non-whitespace character---the rest is English prose in a
\texttt{/*~*/} block. Rosetta files have none, so the model never learned that a real file's first
256 characters are often language-identical boilerplate: it is starved of code, not wrong. \emph{(2) PHP$\to$Rust (16 errors): PHP 8 attributes.} \textbf{13 of the 16} contain \texttt{\#[},
diagnosable straight from training data, where it occurs in \textbf{0.0\% of PHP snippets (0/753)}
but \textbf{9.0\% of Rust (63/703)}---so the model learned \texttt{\#[}$\Rightarrow$Rust. PHP 8
spells attributes identically (\texttt{\#[Group('...')]}) and they appear in \textbf{18.9\% of
held-out PHP (14/74)}, but Rosetta's PHP predates PHP 8. Textbook \emph{distribution
shift}---confidently wrong for a legible reason---and PHP's F1 falls 0.97$\to$0.66.

\textbf{Challenges.} (i) The held-out set took the most work: my first version yielded only 34
snippets/class because the C repositories I picked were too small, so I widened the list, and the
BiGRU's 18$\times$ cost made searching over it impractical. (ii) MPS is non-deterministic, so a fixed seed does not pin the result: eight reruns span
\textbf{91.0--92.7} on test (mean 92.2). Table~3 is the committed run, but the CNN beat the
baseline \textbf{8/8} (worst $+0.87$)---the ranking is robust, the exact figure is not. \textbf{On feasibility}\label{sec:feas}: all data is obtained, the baseline is beaten on both test
sets, and the pipeline reruns in under 3 minutes, so the remaining work---licence-header stripping,
post-2015 PHP sources, SCC \citep{alreshedy2018scc}---is comfortably achievable.
